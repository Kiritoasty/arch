Kiritoasty's Arch Install guide

This is my arch install guide template that is basically like supposed to walk you through the stuff about installing arch (You probably shouldn't use this guide or the arch install guide if you're a noob !! don't do that thats gonna delete your mental health with EVIL mental health

Step 1, Booting the ISO

Boot into the arch iso, I'm assuming that because you use this template, you probably know where to go to get that

Step 2, Confirm UEFI mode

Run this command to figure out if you are on uefi (this is a uefi tutorial !!) 

ls /sys/firmware/efi/efivars

Step 3, Sync package databases

Update the package databases early to avoid EVIL package management errors when you get to installing things like network manager 

pacman -Syy

Step 4, Internet connection

Test your connection with a ping, ethernet usually works automatically !! 

ping -c 3 archlinux.org

If you're using wifi you might wanna use iwctl to connect (Pro tip guys it made me able to install packages without opening up repos with a thing I don't have, called the web browser or web requests) 

iwctl
device list
station wlan0 scan
station wlan0 get-networks
station wlan0 connect your_silly_wifi_name
exit

Step 5, Partitioning the disk

Open cfdisk and select GPT when prompted. First, take a small bit to figure out how to use cfdisk (DO NOT PRESS WRITE INSIDE OF CFDISK IF YOU'RE STILL TRYING TO FIGURE OUT HOW TO USE CFDISK !!), and then do this: Create a 512 Mega Bite (MB/Megabyte) EFI partition and one partition that will be swap that is any amount of gigabytes you want, and a root partition using the remaining space. Write the changes and then go to the quit button when you are done 

cfdisk

Step 6, Formatting

Format the partitions and replace the silly device names if yours are different 

mkfs.fat -F32 /dev/sda1
mkfs.ext4 /dev/sda2

Step 7, Mounting partitions

Mount the root partition first, then mount the EFI partition like you would do with any normal uefi installation 

mount /dev/sda2 /mnt
mkdir -p /mnt/boot
mount /dev/sda1 /mnt/boot

Step 8, Installing the base system so I can boot up my hardware without crying on my laptop keys and then they break from my ocean

Install the base system, kernel, and firmware. 

pacstrap /mnt base linux linux-firmware

Step 9, Generate fstab

Generate the filesystem table so mounts do what they need to do after reboot 

genfstab -U /mnt >> /mnt/etc/fstab

Step 10, Chroot into the degenerate zone (if you are a noob refer to this as the system, /mnt directory which will soon be the / directory)

You are now entering the degenerate zone (the arch linux system chroot, scary) 

arch-chroot /mnt

Step 11, Timezone and locale

Set your timezone and sync the hardware clock 

ln -sf /usr/share/zoneinfo/Region/City /etc/localtime
hwclock --systohc

Enable the locale and generate it, It's Delta 

nano /etc/locale.gen
locale-gen
echo "LANG=en_US.UTF-8" > /etc/locale.conf

Step 12, Hostname

Set a hostname and update the EVIL hosts file 

echo "archlinux" > /etc/hostname
nano /etc/hosts

Step 13, Packages you will need to even have a desktop to use, the thing I made to make this silly install guide

Installing the super delta packages and then enabling them 

pacman -S grub efibootmgr networkmanager sudo nano
systemctl enable NetworkManager

Step 14, Users and passwords

Set the root password, then create a silly regular user Daily driving as root is a bad idea !! do not do that. 

passwd
useradd -m -G wheel user
passwd user
EDITOR=nano visudo

Step 15, Bootloader installation

Install and configure the innocent man grub v0.000001 (whatever the installation of your arch install guide (the iso updates every single second) for UEFI 

grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB
grub-mkconfig -o /boot/grub/grub.cfg

Step 16, Desktop environment (optional but I like being able to click on things so I can program easier without looking at a black and gray screen)

Skip this step if you want a terminal only system btw

XFCE option thing 

pacman -S xfce4 xfce4-goodies lightdm lightdm-gtk-greeter
systemctl enable lightdm

KDE Plasma option !! slightly heavier, but very polished 

pacman -S plasma kde-applications sddm
systemctl enable sddm

Step 17, Finish up your new degenerate distro computer

Exit the chroot, unmount everything, and reboot. this is so you don't boot up into your usb constantly and see that you cannot boot into a existing os Remove the usb guy when prompted. 

exit
umount -R /mnt
reboot
